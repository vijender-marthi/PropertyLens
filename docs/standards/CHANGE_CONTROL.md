# Standards Change Control

PropertyLens standards are protected. Changes to protected standards require explicit approval and review.

## Protected Areas

Changes to any of the following require explicit approval:

- Financial formulas
- API contracts
- Design tokens
- Typography scale
- Number formatting thresholds
- Shared form behavior
- Table behavior
- Domain terminology
- Engine calculations
- Metric Vault snapshot semantics
- Verification classification rules

## Required Standards Change Record

Every protected standards change must include:

- Reason
- Old behavior
- New behavior
- Impacted files
- Migration considerations
- Tests
- Version update when applicable
- Change-log entry
- API compatibility assessment when applicable

## Approval Rule

Do not combine protected standards changes with unrelated UI work. If a requested UI task appears to require formula, engine, or API contract changes, split the work and obtain explicit approval for the standards-impacting change.
